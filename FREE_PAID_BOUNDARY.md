# 無料/有料の境界

この文書は、Agent Memories向けMCP候補について、無料で公開してIssueを集める範囲と、Agent Memoriesユーザー向けに育てる有料/契約者向け範囲を分けるための判断基準です。

## 基本方針

無料版は、使い方を試してもらい、Issueで改善点を集める入口にします。

有料/契約者向けは、Agent Memoriesの価値である「覚える」「積み上がる」「個人に最適化される」を中心にします。

| 投稿前チェック | あり | あり | あり |
| 人間確認ゲート | あり | あり | あり |
| Xテキスト投稿実行 | あり | あり | あり |

| 投稿結果の記録 | なし | あり | あり |
| 反応メモ | なし | あり | あり |
| 次回改善案 | なし | あり | あり |
| 短期投稿履歴を使った改善 | なし | あり | あり |
| 画像付き投稿 | なし | 商品ごとに判断 | あり |
| 動画付き投稿 | なし | 商品ごとに判断 | あり |

見せ方:

- 一般無料版は、規約/API許可範囲内の投稿補助、または本人確認付きのテキスト投稿実行まで。
- Agent Memoriesユーザー特典は、商品によって有料機能の一部または全部を無料開放できる。
- 有料/上位版は、画像、動画、複数媒体、定期投稿、長期分析、記憶連携まで含む。

## 無料で出すもの

無料版に向いている条件:

- その場で完結する
- 記憶保存をしない
- 個人情報や非公開データを入れなくても試せる
- 出力の良し悪しをIssueで説明しやすい
- Agent Memories本体の内部仕様を公開しなくてよい
- 1つのMCPとして小さく実装できる

初回無料候補:

0. Auto Post & Improve Starter MCP
   - テキスト中心のX投稿、note記事、ブログ下書きを作るだけでなく、規約/API許可範囲内の投稿補助まで進める
   - 投稿前チェックと人間確認を挟んだうえで、本人確認付きの投稿実行を候補に含める
   - noteは下書き生成、投稿用整形、投稿手順補助に加えて、自動投稿機能も候補に含める
   - 一般無料は投稿実行までに絞る
   - 投稿URL、投稿結果、反応メモ、次回改善案はAgent Memoriesユーザー特典にする
   - 画像、動画、複数媒体、継続スケジュールは有料版の拡張候補にする
   - 利用者本人のアカウントで、各サービス規約と本人確認の範囲で使う前提にする
   - 初回だけの有料級無料配布として、Issueと認知を集める

1. Reflection Lite MCP
   - 今日の出来事、失敗、改善点を整理する
   - 記憶候補、次回チェック項目、保留すべき項目を出す
   - Agent Memoriesへの誘導が最も自然

2. SNS下書き Lite MCP
   - テーマ、口調、避けたい表現から投稿案を作る
   - SNS別の型、返信案、連投構成、改善メモを出す
   - クリエイター層からIssueを集めやすい

3. ブログ/note構成 Lite MCP
   - メモから見出し案、導入文、CTA案を作る
   - Zenn、Qiita、noteの記事導線と相性がよい
   - 読者像や記事ジャンルの要望を集めやすい

## Issueで集めるもの

無料版では、次のIssueを歓迎します。

- 入力項目が足りない
- 出力の分類が使いにくい
- 口調や文体が合わない
- SNS、ブログ、noteなど媒体別の型がほしい
- 記憶候補として残したい項目が足りない
- Agent Memories契約者向けなら使いたい機能がある
- 公開しても安全な利用例がある

Issueに書かないでほしいもの:

- 個人情報
- 顧客情報
- 非公開の会話ログ
- 認証情報、token、cookie、API key、private key
- 家族、医療、法務、相続など慎重な内容の詳細
- Agent Memories本体やMIRAI Memory本体の非公開仕様を推測させる内容

## 有料/契約者向けにするもの

有料/契約者向けに向いている条件:

- 公式APIや明確に許可された連携方法がある
- 過去の履歴を使う
- ユーザーごとの口調、NG表現、判断基準を覚える
- 繰り返し使うほど出力がよくなる
- 非公開メモ、個人設定、長期エピソードを扱う
- Agent Memories本体と連携して記憶を昇格、整理、再利用する
- 外部サービス連携や継続運用が必要

有料/契約者向け候補:

- Auto Post & Improve Pro
  - 投稿履歴、反応、口調、NG表現、販売導線を記憶する
  - 画像付き投稿、動画付き投稿を扱う
  - 複数媒体への同時投稿やシリーズ投稿を扱う
  - 定期投稿、承認フロー、改善ループ、媒体別最適化を扱う
  - 投稿後の反応をAgent Memoriesに保存し、次回の投稿案へ反映する
  - 複数アカウント運用や外部サービス連携は、規約と誤投稿リスクを確認した範囲で提供する

- Agent Memories User Posting Pack
  - Agent Memoriesユーザーには、一般無料より広い機能を無料特典として開放する
  - 投稿URL、投稿結果、反応メモ、次回改善案を残す
  - Agent Memories登録の理由になる「投稿して記録・改善できる」体験を提供する

- Reflection Pro
  - 日次振り返りを継続記憶へ昇格する
  - 失敗パターン、改善ルール、次回チェックを蓄積する

- SNS Memory Pro
  - 過去の投稿反応、口調、NG表現、伸びたテーマを反映する
  - 投稿前後の改善メモを継続管理する

- Blog/note Memory Pro
  - 過去記事、読者像、継続テーマ、CTA方針を記憶する
  - シリーズ記事や商品導線に合わせて構成する

- キャラクター記憶設定MCP
  - AIキャラクターの口調、関係性、重要な出来事、避けたい表現を整理する
  - 長期利用で価値が増えるため、契約者向けの中核候補にする

- 家族メモリーMCP / 終活メモリーMCP
  - 思い出、残したい言葉、手続きメモを整理する
  - 医療、法務、相続判断の代替にはしない
  - 慎重な扱いが必要なため、無料公開ではなくガードレール付きで検討する

## 実装優先順位

最初の実装順:

0. Auto Post & Improve Starter MCP
1. Reflection Lite MCP
2. SNS下書き Lite MCP
3. ブログ/note構成 Lite MCP

Auto Post & Improve Starterを初回無料で出す理由:

- 投稿作成だけでは無料AIと差別化できない
- 規約/API許可範囲内の投稿補助、または本人確認付きの投稿実行までを一般無料にする
- 投稿後記録と次回改善はAgent Memoriesユーザー特典にして、登録理由を作る
- APIなし/許可不明の実行部分は無料OSS/検証用であり、価値訴求の中心にしない
- クリエイター、個人事業主、副業ユーザーに刺さりやすい
- Issueで、媒体別投稿フロー、投稿後記録、改善メモ、承認フロー、危険な自動化の境界について要望を集めやすい
- 無料でも強く見せつつ、有料版には「覚えて伸ばし続ける」価値を残せる

Reflection Liteを次に実装する理由:

- Agent Memoriesの本質に最も近い
- 無料で試した後に「これを記憶として残したい」という導線が作りやすい
- Issueで、記憶分類、昇格基準、振り返り項目の改善要望を集めやすい

## 自動投稿の公開境界

自動投稿は、全面公開する範囲と価値訴求を慎重に分ける。

有料化しやすい自動化:

- 公式APIがある
- サービス規約上、外部ツール連携や自動投稿が許可されている
- 認証方法を安全に扱える
- 提供者側が仕様、制限、責任範囲を説明できる
- 承認フロー、履歴管理、改善分析、Agent Memories連携として価値を積める

無料OSS/実験ツールに寄せる自動化:

- 公式APIがない
- 自動化が明確に許可されていない
- サービス側の仕様変更で壊れやすい
- 利用者本人の判断と責任で使う必要がある
- 有料商品として売ると、提供者側の信用・法務・規約リスクが重くなる
- LP、料金表、販売ページで「有料の売り」として見せない

無料で公開できる範囲:

- テキスト投稿本文の生成
- 投稿前チェック
- 投稿用データの整形
- 人間確認ゲート
- Xのテキスト投稿補助、または本人確認付きの投稿実行
- note向けの下書き生成、投稿用整形、投稿手順補助
- noteのテキスト中心の自動投稿機能
- 単発または少数投稿の運用
- 利用者本人のアカウントで規約に沿って使うための注意書き

Agent Memoriesユーザー特典として無料開放できる範囲:

- 投稿URLの記録
- 投稿結果の記録
- 反応メモの記録
- 次回投稿の改善案
- 短期の投稿履歴を使った改善

無料で出しすぎない範囲:

- 画像付き投稿
- 動画付き投稿
- 複数媒体への同時投稿
- 投稿素材の自動生成と投稿
- ログイン突破、reCAPTCHA回避、凍結回避、制限回避
- 認証情報、cookie、session情報の保存や共有
- スパム投稿、大量投稿、なりすまし投稿
- 複数アカウントの定期自動投稿
- 長期の投稿履歴からの継続改善
- 高度な反応分析の記憶化
- 個人ごとの口調、NG表現、販売導線の長期保存

判断:

一般無料の価値は「投稿を作る」だけではなく、「規約/API許可範囲内の投稿補助、または本人確認付きのテキスト投稿実行まで進められる」こと。

Agent Memoriesユーザー特典は「投稿して記録・改善できる」。

有料/上位版は「画像・動画・継続運用・高度な記憶・分析で伸ばし続ける」。

APIなし/許可不明の自動投稿は、有料商品として売らず、無料OSS/実験ツールとして集客とIssue収集に使う。

公式API/許可済みの自動投稿は、有料版や法人向け運用に載せやすい。

固定表現:

- 各サービス規約・API許可範囲内で利用してください。
- 非公式自動化は販売対象外です。
- APIなし/許可不明の実行部分は、無料OSS/検証用であり、非保証です。

避ける表現:

- 非公式自動化に対応できます
- APIなし投稿を有料機能として提供します
- 規約確認なしで自動投稿できます
- 凍結や制限を回避できます

## Miraigentとの切り分け

Miraigent:

- 企業向け
- 公式API、規約準拠、承認フロー、安全運用を中心にする
- 有料導入、無料診断、法人向け運用設計につなげる
- APIなし/許可不明の自動投稿を有料商品として前面に出さない
- 非公式自動化をLPや料金表の訴求軸にしない

Agent Memories / OSS無料ツール:

- 個人クリエイター、個人事業主、副業ユーザー向け
- APIなし/許可不明の投稿補助、自動投稿実験、便利ツールを無料OSS/検証用として出す
- Issueで不具合、要望、媒体別の使い方、危険な自動化の境界を集める
- 利用は各サービス規約と本人判断の範囲で行うことを明記する
- 非保証、自己責任、販売対象外の境界を明記する

Agent Memories有料:

- 記憶、改善、履歴、口調、NG表現、勝ちパターン保存に課金価値を置く
- 画像、動画、複数媒体、定期投稿、反応分析などの拡張機能を有料価値にする
- 外部投稿実行そのものは、公式APIや明確に許可された方法があるものを中心にする
- APIなし/許可不明の実行部分は、有料価値の中心にしない

## 判断メモ

機能の量で無料/有料を分けない。

無料は「試して、Issueを出せるもの」。

有料は「覚えて、積み上がり、個人に合っていくもの」。
